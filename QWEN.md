# Anti-Fraud Shield — Проектный контекст

## 📌 Обзор проекта

**Anti-Fraud Shield** — браузерная мини-игра на Canvas + TypeScript, моделирующая работу оператора мониторинга банковских транзакций в реальном времени. Игрок классифицирует падающие карточки транзакций: блокирует мошеннические и пропускает легитимные.

**Ключевая особенность:** полностью безынтерфейсная коммуникация — вся информация передаётся через иконки, цвета и пространственную логику без текстовых подписей.

**Целевая аудитория:**
- Новые сотрудники банка (обучение распознаванию фрод-паттернов)
- Опытные аналитики (тренировка, соревнование)
- Руководители/коллеги (демонстрация сложности работы антифрод-системы)

## 🏗 Технологический стек

| Компонент | Технология |
|-----------|-----------|
| Сборщик | **Vite** (dev server, HMR, оптимизация ассетов) |
| Язык | **TypeScript** (строгий режим, статическая типизация везде) |
| Рендер | **Canvas API** (игровое поле) + **DOM** (HUD, меню, оверлеи) |
| Хранение | **localStorage** (лидерборд, настройки, прогресс) |
| Зависимости | **Нативные веб-стандарты** (без внешних библиотек) |

### Архитектурные принципы

- **Модульность** — каждая система инкапсулирует свою зону ответственности
- **Шина событий** — системы общаются через строго типизированную шину, без прямых зависимостей
- **Пул объектов** — переиспользование карточек, нулевая нагрузка на GC
- **Конечный автомат** — чёткое разделение режимов, защита от недопустимых переходов
- **Слоистая архитектура** — Canvas = игровое поле, DOM = UI-слой, централизованный z-index

### Планируемая структура src/

```
src/
├── assets/       # Иконки, звуки, шрифты
├── types/        # Глобальные интерфейсы и перечисления
├── core/         # Игровой цикл, состояния, события, таймер
├── entities/     # Карточки транзакций, пул, спавнер
├── systems/      # Ввод, сложность, рендер, очки, дуэль
├── storage/      # Лидерборд, localStorage
├── ui/           # Меню, HUD, оверлеи, экраны
├── config/       # Тема, правила генерации, сценарии обучения
├── utils/        # Математика, валидация, хелперы
└── main.ts       # Точка входа
```

## 📅 Этапы реализации

### Этап 1: MVP + Обучение
1. Инициализация проекта (Vite + TypeScript)
2. Пул объектов, спавнер, система ввода
3. Таймер с визуализацией и логикой завершения
4. Режим обучения: сценарии, подсказки, прогресс
5. HUD, движок сложности, визуальный отклик
6. Мобильная адаптация, стабилизация
7. Тестирование «без подписей»

### Этап 2: Лидерборд + Дуэль
1. Хранилище рекордов (CRUD, валидация, фильтрация)
2. Экран лидерборда (сортировка, личный рекорд, экспорт)
3. Режим дуэли (переключение игроков, синхронизация раундов)
4. Экран сравнения, анимация победы, серия раундов
5. Финальные экраны, тестирование потоков, релиз

## 📝 Примечания для разработки

- **Рабочее окружение:** Windows (использовать соответствующие команды)
- **Язык комментариев в коде:** русский
- **Строгая типизация:** все данные, контракты и состояния через интерфейсы и enum
- **Без внешних зависимостей:** только нативные веб-стандарты
- **Все цвета/радиусы/тени:** CSS Custom Properties для темизации
- **Тouch-зоны:** ≥ 44×44px
- **prefers-reduced-motion:** полное отключение анимаций
