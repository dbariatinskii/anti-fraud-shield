# 🎨 Требования к UI (Anti-Fraud Shield)

## 🌐 Глобальные принципы интерфейса
- **Визуальная автономность:** Интерфейс должен функционировать при полном отсутствии текста. Все значения, состояния и действия передаются через иконки, цвета, формы и пространственное расположение.
- **Цветовая семантика:** Фиксированная палитра без декоративных отклонений. Зелёный = безопасное состояние/рост, красный = риск/ошибка/критический таймер, синий/серый = нейтральные данные, жёлтый/оранжевый = фокус/предупреждение.
- **Иконографика:** Используются только устоявшиеся банковские и процессные паттерны. Каждая иконка проходит проверку на однозначное чтение без контекста.
- **Сетка и ритм:** Базовая модульная сетка 8px. Все отступы, радиусы и размеры кратны базовому модулю. Зоны нажатия строго ≥ 44×44px.
- **Иерархия внимания:** Игровое поле > Критические индикаторы (щит, таймер) > Второстепенные данные (очки, прогресс) > Фоновые элементы. Никакой визуальный шум не конкурирует с карточками транзакций.

## 🖥 Требования по экранам

### Главное меню
- Три крупные кликабельные области с доминирующими иконками режимов
- Анимация при наведении/фокусе: лёгкое поднятие, усиление тени, плавная смена акцента
- Фоновая абстракция: сетка или поток линий, не отвлекающая от навигации
- Кнопка рекордов вынесена отдельно, визуально отделена от режимов игры

### Игровой HUD
- Верхний левый угол: индикатор щита с плавным заполнением, меняет оттенок при критическом уровне
- Верхний правый угол: счётчик очков (минималистичный шрифт) + круговой таймер с цветовой градацией
- Нижний край поля: тонкая линия или градиентная зона, обозначающая точку принятия решения
- Прозрачность фона HUD ≤ 10%, чтобы не перекрывать игровое поле

### Режим обучения
- Оверлей с мягким затемнением краёв, центральный фокус на активной карточке
- Подсказка появляется при наведении/удержании: иконка паттерна + краткое пояснение
- Кнопки управления подсказками вынесены за пределы игрового поля, не перекрывают карточки
- Прогресс-бар внизу экрана с визуальными маркерами этапов

### Экран завершения
- Центрированная карточка с итогами: очки, точность, время
- Поле ввода имени (опционально) с валидацией длины и символов
- Кнопки действий расположены горизонтально, визуально разделены
- Анимация появления результатов каскадная: щит → очки → точность → кнопки

### Таблица рекордов
- Список/таблица с фиксированной шириной колонок, скролл только при необходимости
- Табы фильтрации в виде переключателей с индикатором активного
- Личный рекорд подсвечен фоном или рамкой, вынесен в начало списка
- Кнопка экспорта/шеринга в виде иконки, не конфликтует с основной навигацией

### Режим дуэли
- Индикатор текущего игрока: крупная пиктограмма + цветовая пульсация в углу
- Экран сравнения: два блока результатов рядом, визуальная линия победы/поражения
- Кнопка перехода к следующему раунду центрирована, доступна только после отображения итогов

## ⚡ Взаимодействие и состояния элементов
- **Отклик:** Визуальная реакция на действие < 100 мс. Ripple, scale или цветовая вспышка при клике/тапе.
- **Состояния:** Каждый интерактивный элемент имеет чёткие состояния: Default, Hover/Focus, Active, Disabled, Error/Success.
- **Переходы:** Смена экранов через fade или slide 200–300 мс. Резкие скачки и моргание запрещены.
- **Пауза:** Затемнение фона 60%, центрированная иконка паузы, кнопки «Продолжить» и «Выход». Таймер останавливается мгновенно.
- **Обратная связь при ошибках:** Экран/зона кратковременно окрашивается в акцентный цвет, щит теряет часть заполнения, карточка-ошибка исчезает с анимацией распада. Без модальных окон.

## ♿ Доступность и адаптивность
- **Контраст:** Минимум 4.5:1 для всех текстовых и иконографических элементов относительно фона. Проверка в оттенках серого обязательна.
- **Навигация:** Логичный `tabindex`, видимый фокус, поддержка клавиатуры (Enter/Space — выбор, Esc — пауза/назад, стрелки — навигация по меню).
- **Мобильная адаптация:** Отказ от горизонтального скролла, адаптивное масштабирование HUD, touch-зоны ≥ 44px, жесты не конфликтуют с системными.
- `prefers-reduced-motion`: Полное отключение анимаций, статичные переходы, мгновенные состояния.
- **Шрифты:** Системный стек, относительные единицы (rem), поддержка расширения текста на 30% без поломки вёрстки.

## 🛠 Технические требования к реализации
- **Слоистая архитектура:** Canvas рендерит игровое поле, DOM отвечает за HUD, меню и оверлеи. Z-index управляется централизованно, конфликты наложения исключены.
- **Темизация:** Все цвета, радиусы, тени и отступы вынесены в CSS Custom Properties. Смена темы происходит без перезагрузки.
- **Оптимизация рендера:** `will-change` только для активно анимируемых элементов, CSS `contain` для изоляции перерисовок, ленивая загрузка ассетов.
- **Интеграция с TypeScript:** Строгая типизация DOM-узлов, type-safe обработчики событий, валидация данных перед инъекцией в шаблон.
- **Кросс-браузерность:** Fallback для старых движков, проверка корректности отображения в Safari, Chrome, Edge, Firefox (desktop + mobile).
- **Подготовка к локализации:** Все строки вынесены в словарь, UI спроектирован с запасом ширины для языков с длинными словами.

## ✅ Валидация UI/UX

| Проверка | Метод |
|----------|-------|
| **Скриншот-тест** | Коллега из смежного отдела за 3 секунды называет продукт и механику без пояснений |
| **Слепой тест** | Игрок проходит базовый цикл, ориентируясь только на цвета, формы и расположение |
| **Контраст и дальтонизия** | Проверка через симуляторы протанопии/дейтеранопии, сохранение читаемости статусов |
| **Мобильный сценарий** | Удобство управления одной рукой, отсутствие случайных нажатий, стабильный FPS |
| **Производительность UI** | Падение FPS ≤ 5 при одновременной анимации Canvas и DOM-оверлеев |
| **Доступность** | Прохождение axe DevTools или Lighthouse Accessibility ≥ 95 баллов |

## 🏦 Визуальные якоря продукта (обязательно к соблюдению)
- Щит = запас безопасности / допустимый уровень риска
- Падающие карточки = поток транзакций в реальном времени
- Цветовая маркировка = скоринговое решение системы
- Нижняя граница поля = точка фиксации/отправки решения в ядро
- Круговой таймер = окно реакции оператора до авто-эскалации
- Прогресс-бар обучения = накопление экспертизы до допуска к боевому режиму

Эти требования гарантируют, что интерфейс останется интуитивным, профессиональным и готовым к интеграции в корпоративные процессы без дополнительной документации.
